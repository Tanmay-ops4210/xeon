Step 4: Set Storage Policies (Optional)

For each bucket, you can set up Row Level Security policies in the SQL Editor:

-- Public read access CREATE POLICY "Public read access" ON storage.objects FOR SELECT USING (bucket_id = 'bucket-name'); -- Authenticated upload access CREATE POLICY "Authenticated upload access" ON storage.objects FOR INSERT WITH CHECK (bucket_id = 'bucket-name' AND auth.role() = 'authenticated'); -- Authenticated update access CREATE POLICY "Authenticated update access" ON storage.objects FOR UPDATE USING (bucket_id = 'bucket-name' AND auth.role() = 'authenticated'); -- Authenticated delete access CREATE POLICY "Authenticated delete access" ON storage.objects FOR DELETE USING (bucket_id = 'bucket-name' AND auth.role() = 'authenticated');

Replace 'bucket-name' with the actual bucket name (event-images, profile-pictures, etc.)

Step 5: Verify Setup

After creating all buckets, run this command to verify they exist:

node setup-storage-buckets-simple.js
